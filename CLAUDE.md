# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

Personal dotfiles/server configuration repo. Two entry points:

- **`setup.sh`** — run locally to symlink configs into `$HOME` and install CLI tools (glow, nvim, node, uv, Claude Code, Codex, etc.).
- **`deploy.sh`** — run locally to bootstrap a remote server over SSH (authorize SSH key, copy `gh`/Claude/Codex auth, clone this repo on the remote, then invoke `setup.sh` there).

Shared helpers (`run_step`, `retry`, `backup_and_link`, `backup_and_copy`) live in `lib/common.sh` and are sourced by both scripts plus `install_claude_plugins.sh`.

## Architecture

- **Flat repo structure**: top-level files are the product (shell configs, editor configs, tool settings). Scripts live alongside them. No `src/` or `tests/` tree.
- **Error handling**: scripts use `set -uo pipefail` (no `-e`). Failures are tracked per-step via `run_step()` into a `FAILURES=()` array. The script continues and reports all failures at the end.
- **Paths**: all paths inside scripts are relative to `$DIR` (derived via `$(cd "$(dirname "$0")" && pwd)`). Never hardcode absolute paths.
- **Generated compat configs**: `~/.server-configs-generated/` holds version-adaptive variants regenerated by `setup.sh` on every run:
  - `tmux.compat.conf` — terminal type, true color, passthrough
  - `vimrc.compat` — clipboard, listchars, Neovim detection
  - `gitconfig.compat` — credential helper
  - `bashrc_compat` — tool-specific aliases with version-appropriate flags
  - **Never edit generated files directly** — edit the `render_*()` functions in `setup.sh`.

## Conventions

- **Symlinks** for most configs via `backup_and_link(src, dst)`. **Copies** for secrets/tool configs via `backup_and_copy(src, dst)` (allows local overrides without affecting the repo).
- Existing files are backed up as `.bak` before replacing. `uninstall.sh` restores them.
- **Idempotency**: every install function checks if the tool exists before installing. Use `--force` to reinstall. Re-running `setup.sh` is always safe.
- **Architecture detection**: install functions handle `x86_64` and `aarch64`. Unsupported architectures skip gracefully.
- `retry()` wraps network calls (3 attempts, 2-second delay).

## Config Relationships

```
~/.bashrc
  └── sources ~/.bashrc_exports  (symlink → bashrc_exports)
       └── sources ~/.server-configs-generated/bashrc_compat
  └── sources ~/.bashrc_aliases  (symlink → bashrc_aliases)
       └── sources ~/.server-configs-generated/bashrc_compat

~/.vimrc  (symlink → vimrc)
  └── sources ~/.server-configs-generated/vimrc.compat

~/.tmux.conf  (symlink → tmux.conf)
  └── sources ~/.server-configs-generated/tmux.compat.conf

~/.gitconfig  (symlink → gitconfig)
  └── includes ~/.server-configs-generated/gitconfig.compat
```

## Setup

```bash
./setup.sh               # idempotent — re-run is safe
./setup.sh --force       # reinstall CLI tools even if present
```

## Remote deploy

```bash
./deploy.sh              # interactive: pick steps, enter host
./deploy.sh --yes        # accept defaults, skip confirmations
```

Claude and Codex copy only known auth files (`~/.claude/.credentials.json`, `~/.codex/auth.json`) from the local machine to the remote. GitHub CLI auth is recreated from the local `gh auth token` before cloning, with `hosts.yml` used only as a plaintext-token fallback. Keychain-backed Claude auth is reported as non-transferable and must be set up on the remote with `claude auth login` or `claude setup-token`. Auth material is **never** committed to this repo (see `.gitignore`).

## Vim & Neovim

- **vimrc** is plugin-free (uses only built-in vim features + netrw). Don't add plugin managers or plugin dependencies.
- **nvim/** is the Neovim config (symlinked to `~/.config/nvim/`). Uses lazy.nvim for plugins.
  - Layout: `init.lua` → `lua/config/{options,keymaps,autocmds,lazy}.lua` → `lua/plugins/*.lua`
  - Leader is Space.
  - 12 plugin spec files in `lua/plugins/`. Each file = one plugin or group.
  - Which-key groups: `<leader>f` = find, `<leader>g` = git, `<leader>b` = buffer.
  - **Oil replaces netrw** (`vim.g.loaded_netrw = 1`). `<leader>e` and `-` open Oil.
  - **`<leader>m` differs between vim and nvim**: vim runs `!glow %` (terminal); nvim triggers `MarkdownPreviewToggle` (browser) on markdown files.
  - After editing plugin specs, regenerate the lock: `nvim --headless '+Lazy! sync' +qa`.
  - Setup installs nvim via (in order): `module load`, AppImage (skipped on glibc < 2.28), tarball to `~/.local/`.
- **tmux prefix** is `Ctrl-a` (not the default `Ctrl-b`).
- **Indentation**: vimrc defaults to 4-space tabs; web filetypes (html/css/js/ts/json/yaml) use 2-space via autocmd. Neovim adds `lua` to the 2-space list.
- Glow is used for markdown preview in vim (`<leader>m`).

## CHPC (Center for High Performance Computing)

`is_chpc()` in `lib/common.sh` detects CHPC systems via hostname (`*.chpc.utah.edu`) or path (`/uufs/chpc.utah.edu`). When on CHPC, `setup.sh` automatically:

- Uses `module load` for gh, Node.js, uv, btop, Claude Code, and Codex instead of self-installing
- Generates CHPC-safe agent settings even if the CLIs are not currently installed or loaded
- Uses approval-required, sandboxed defaults (`default` mode for Claude, `untrusted`/`workspace-write` for Codex)
- Skips MCP server installation by default (`install_claude_plugins.sh` — needs CHPC approval first)

**Module name placeholders** — update in `setup.sh` after confirming on CHPC:

```bash
CLAUDE_MODULE_CANDIDATES=("claude-code" "claude")   # check with: module spider claude
CODEX_MODULE_CANDIDATES=("codex" "openai-codex")    # check with: module spider codex
GH_MODULE_CANDIDATES=("gh")                          # check with: module spider gh
NODE_MODULE_CANDIDATES=("nodejs")                     # check with: module spider nodejs
UV_MODULE_CANDIDATES=("uv")                           # check with: module spider uv
BTOP_MODULE_CANDIDATES=("btop")                       # check with: module spider btop
```

The CHPC-safe configs are written to `~/.server-configs-generated/` (not the repo files). Run `setup.sh` on CHPC to regenerate them. After MCP approval, run `install_claude_plugins.sh --allow-chpc` or set `SERVER_CONFIGS_ALLOW_CHPC_MCP=true`.

## Do Not

- **Do not** add plugins to `vimrc` — it is intentionally plugin-free.
- **Do not** edit files in `~/.server-configs-generated/` — they are overwritten by `setup.sh`.
- **Do not** hardcode absolute paths in scripts — use `$DIR`.
- **Do not** use `set -e` in scripts — `run_step` handles errors.
- **Do not** commit auth files — see `.gitignore`.
- **Do not** unload `chpc/1.0` if running on CHPC — it is a sticky module required for the environment.

## Lint / pre-commit

A pre-commit hook at `.githooks/pre-commit` runs `bash -n` on staged shell files, runs warning-level `shellcheck` if available, and scans staged diffs for common secret patterns. It is Bash 3.2-compatible for macOS control hosts. `setup.sh` wires this via `git config core.hooksPath .githooks` on first run.
